#### **Docsify <span class="tab-badge-blue">本站</span>**

+ 📃 项目介绍  +
  
  #### 一个神奇的文档站点生成器。
  
  - 无需构建静态HTML文件，直接将Markdown文件转换为网站；<br>
  - 轻量级设计，使用简单；<br>
  - 支持全文搜索插件、多主题切换和插件扩展；<br>
  - 原生支持Emoji显示；<br>
+ 👀 预览 +
  
  ![1.png](https://raw.githubusercontent.com/Aleeyoo/note-gen-image-sync/main/53c53ec9-b4b1-464d-8fcc-901b5b3ab447.png)
 
+ 🔗 相关链接 +
  
  1. 官方网站：[docsify.js.org](https://docsify.js.org/#/)<br>
  2. 部署流程：[部署文档](https://docsify.js.org/#/zh-cn/deploy)<br>
  3. 快速启动模板：[Docsify Template](https://github.com/docsifyjs/docsify-template)<br>
  4. 命令行工具：[Docsify CLI](https://github.com/docsifyjs/docsify-cli)<br>
  5. CDN资源：UNPKG / jsDelivr / cdnj<br>
  6. 社区讨论：[Discord](https://discord.com/invite/3NwKFyR)<br>
+ 🤔 优点与缺点 +
  
  #### 👍 优点
  
  1. 部署简单：本地运行 `npm i docsify-cli -g`、`docsify init ./docs`,将生成文件上传 GitHub 即可；<br>
  2. 插件丰富：具有多种插件，如全文搜索、多主题切换、代码高亮等，引用方式简单；<br>
  3. 自定义布局：支持自定义布局，可根据需求定制网站、插件外观；<br>
  4. 支持 Markdown 语法：无需学习新的语法，直接使用 Markdown 编写文档；<br>
  5. 支持自定义侧边栏导航显示标题，且支持自定义 `subMaxLevel: 1`；<br>
  
  #### 👎 缺点
  
  1. 插件市场版本较老，部分插件可能失效；<br>
  2. 需要手动添加目录 `_sidebar.md` 来配置侧边栏导航；<br>
  3. 不支持在线编辑，需要上传到 GitHub Repository，通过 GitHub Actions 才能生效，具有一定的时间延迟；<br>
  4. 部署到 GitHub Pages 后，网络延迟较高，加载速度较慢；<br>
+ 🧐 个人评价 +
  
  > [!NOTE]
  > 个人实践后的片面评价
  
  &emsp;&emsp;首先，一分钱不花，部署到 GitHub Pages 后，如果不自定义域名，可能会有访问速度的问题。这个项目部署难度不高，但难在想要自定义出满意的文档站点，需要一定的前端基础，或者借助 AI，总要花费一定的时间。并且，除了基础的 Markdown 编写的，也要会使用一些 HTML 标签辅助写作。<br>
  &emsp;&emsp;关于使用 Git 上传不方便，也可以使用一些存储到 GitHub 的笔记软件，比如 [Notegen](https://notegen.top/cn) 来写文章。<br>
  &emsp;&emsp;总的来说，有时间来练练也不错，当作个人文档站点、产品说明，甚至个人博客都不错。<br>

#### **BroadcastChannel**

+ 📃 项目介绍  +
  
  #### 将您的 Telegram 频道变成微博。
  
  - 将Telegram Channel转换为可自定义的微博客（MicroBlog）；<br>
  - 支持部署在Cloudflare、Netlify、Vercel等Serverless平台（需Node.js SSR支持）或VPS；<br>
+ 🔗 相关链接 +
  
  1. GitHub仓库：[ccbikai/BroadcastChannel](https://github.com/ccbikai/BroadcastChannel)<br>
  2. 演示页面：[面条实验室](https://memo.miantiao.me/)<br>
  3. 中文文档、教程：[ccbikai/BroadcastChannel](https://github.com/ccbikai/BroadcastChannel/blob/main/README.zh-cn.md)<br>
+ 👀 预览 +
  
  ![image.png](https://raw.githubusercontent.com/Aleeyoo/note-gen-image-sync/main/75c16bd6-6853-42a2-805b-9a278bf28858.png)

+ 🤔 优点与缺点 +
  
  #### 👍 优点
  
  1. 部署简单，通过环境变量自定义基础信息；<br>
  2. 支持 Telegram 富文本格式（如加粗、列表、链接等）；<br>
  3. 可注入 HTML 代码，自定义修改页头、页脚等局部样式；<br>
  4. 图片直接使用 Telegram 频道内的素材，无需额外上传至其他平台；<br>
  5. 支持 Rss 订阅，可使用 [rss.beauty](https://rss.beauty/) 美化。<br>
  
  #### 👎 缺点
  
  1. 撰写依赖 Telegram 频道，需借助科学上网工具访问 Telegram；<br>
  2. 仅支持公开 Telegram 频道，私有频道无法实现文章同步；<br>
  3. 内容整体样式由模板固定，自定义程度中等；<br>
+ 🧐 个人评价 +
  
  > [!NOTE]
  > 个人实践后的片面评价
  
  &emsp;&emsp;如果你使用 Telegram 较多，那么这个项目首推，既可以当朋友圈发日常（可发视频、图片、文件）。也可以作为文件分享平台，分享一些内容，让用户不用登录 Telegram 即可下载一些文件。<br>

#### **Gmeek**

+ 📃 项目介绍  +

#### Gmeek is a Blog All in Github.

- 基于GitHub Pages、Issues和Actions的超轻量级博客模板；<br>
- 无需本地部署，18秒3步完成搭建（创建仓库→启用Pages→通过Issues写作）；<br>
- 自动触发Actions部署，支持标签分类管理；<br>

+ 🔗 相关链接 +
  
  1. Demo页面：[blog.meekdai.com](https://blog.meekdai.com/)
  2. Gmeek更新日志：[Gmeek更新日志](https://blog.meekdai.com/post/Gmeek-geng-xin-ri-zhi.html)
  3. 快速上手教程：[快速上手教程](https://blog.meekdai.com/post/Gmeek-kuai-su-shang-shou.html)
  4. 通过模板创建仓库：[通过模板创建仓库](https://github.com/new?template_name=Gmeek-template&template_owner=Meekdai)
  
+ 👀 预览 +
  
  ![image.png](https://raw.githubusercontent.com/Aleeyoo/note-gen-image-sync/main/ad9cc3f8-68d1-4289-9f55-4e96b9aad83a.png)

+ 🤔 优点与缺点 +
  
  #### 👍 优点
  
  1. 支持通过插件扩展功能；<br>
  2. 修改官方提供的 JSON 配置文件，调整主题风格、导航菜单等基础样式；<br>
  3. 支持标签分类管理，通过标签筛选文章；<br>
  4. 提供 Rss 订阅。<br>
  
  #### 👎 缺点
  
  1. 通过 GitHub Issue 提交文章,需为文章添加特定 Label 才能显示；<br>
  2. 图片需通过特定 HTML 格式附加；<br>
+ 🧐 个人评价 +
  
  > [!NOTE]
  > 个人实践后的片面评价
  
  &emsp;&emsp;中规中矩，适合写好后复制粘贴到 issue 发布，但默认 Markdown 图片显示不生效，用特定语法。<br>

#### **Tinymind**

+ 📃 项目介绍  +
  
  #### Tinymind - 将您的博客和想法与 GitHub 同步。
  
  - 一键授权GitHub，将仓库作为博客/笔记数据存储；<br>
  - 自动创建tinymind-blog仓库，内容实时同步至GitHub；<br>
  - 支持Markdown编辑、图片拖拽上传及长文本折叠（“Show More/Less”）；<br>
  - 生成个人主页（如：tinymind.me/mazzzystar）；<br>
+ 🔗 相关链接 +
  
  1. 官网：[tinymind.me](https://tinymind.me/)<br>
  2. 演示页面：[tinymind.me/mazzzystar](https://tinymind.me/mazzzystar)<br>
  3. GitHub仓库：[mazzzystar/tinymind](https://github.com/mazzzystar/tinymind)<br>
  4. macOS客户端：[Tinymind.dmg](https://github.com/mazzzystar/tinymind/releases/download/v0.1.0/Tinymind.dmg)<br>
+ 👀 预览 +
  
  ![image.png](https://raw.githubusercontent.com/Aleeyoo/note-gen-image-sync/main/f9bb73a2-f40f-4b8c-a289-ff0a8581a3be.png)

+ 🤔 优点与缺点 +
  
  #### 👍 优点
  
  1. 提供网页端可视化编辑器，可直接在线撰写；<br>
  2. 默认使用官方二级域名，支持绑定自定义域名；<br>
  3. 图片可直接粘贴上传（系统自动保存至 GitHub 仓库）；<br>
  4. 页面简洁；<br>
  
  #### 👎 缺点
  
  1. 国内无法直接访问官网，需要通过代理；<br>
  2. 不支持 HTML 语法；<br>
  3. 无插件扩展机制，功能扩展性较弱；<br>
  4. 自定义选项极少，主题风格、导航栏、页面布局等界面元素均无法调整；<br>
  5. 图片的删除 / 修改需手动进入 GitHub 仓库操作；<br>
+ 🧐 个人评价 +
  
  > [!NOTE]
  > 个人实践后的片面评价
  
  &emsp;&emsp;这个项目最大优点就是可以在线编辑、上传图片到 GitHub 仓库，且页面、操作简单。但问题较严重，无法直连（满足想让人看见，又不想让太多人看见），网站图标也不能自定义，网址路径一团糟。个人评价为将一个笔记软件搬到网上而已。<br>
